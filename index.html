História da Web

Primeiros Computadores

(Voltar)

No início, os comuptadores eram gigantescos, o "portátil" da época era conseguir arrastar o computador de uma sala para a outra. Com Bill Gates, Steve Jobs e outros grandes nomes, o comuptador passou a ser muito mais acessível para o público em geral.

Como Surgiu a Internet

(Voltar)

O impulso de desenvolver a comunicação existe desde sempre. As primeiras sementes da web como conhecemos hoje foram plantadas com comunicações para fins militares, principalmente durante a Guerra Fria. Após vários avanços, Tim Berners-Lee criou o primeiro navegador, chamado WorldWideWeb.

Grandes Nomes

(Voltar)
  • Grace Hopper: Criadora da linguagem de alto nível Flow-Matic.
  • Joseph Carl Robert Licklider: plantou a semente da comunicação entre dois pontos distintos através do computador
  • Robert E. Kahn: Apresentou em público o primeiro e-mail e criou o protocolo TCP/IP.
  • Tim Berners-Lee: Inventor do WWW; popularizou o HTTP e o HTML.
  • Marc Andreessen: Tornou o navegador amigável, com mais recursos gráficos.
  • Bill Gates: fundador da Microsoft, que criou o Internet Explorer. Popularizou muito o computador, e com isso, a internet.
  • Steve Jobs: liderava a Apple quando esta popularizou o smartphone, o tornando muito mais amigável para o uso de pessoas comuns.

O que são Clients

Clientes no Linguajar Web

(Voltar)

É o que consome alguma informação fornecida por um servidor. Quando alguém acessar meu site, o navegador que a pessoa está usando se torna o cliente do servidor que hospeda os dados do meu site.

Navegadores

(Voltar)

Programa instalado no computador que interpreta os arquivos mandados pelo servidor (HTML, CSS, JS...) Seguem padrões W3C, que é um conselho de definição dos comportamentos dos navegadores. Às vezes, um navegador pode ser incompatível com algum recurso HTML ou de outra tecnologia/linguagem. O site W3 Schools detalha isso.

Aplicações Web

(Voltar)

O que são? Se tratam de soluções que fazem a comunicação entre Client e Server para que o usuário possa aproveitar as funcionalidades, sem precisasr instalar um app no seu dispositivo; YouTube é uma aplicação web e um app ao mesmo tempo; mesma coisa Facebook, Instagram...

Dispositivos Móveis

(Voltar)

Qual a importância de falar sobre isso? As pessoas cada vez mais acessam conteúdos na internet através do celular. A maioria acessa pelo celular, na verdade, por isso é importante priorizar (sim, mais importante que desktop) a exibição dos conteúdos em dispositivos mobile, para uma boa experiência de usuário.


O que são Servers

Sobre Servidores

(Voltar)

São máquinas físicas para armazenamento e distribuição de recursos importantes para as máquinas que "dependem" desse servidor, os Clients.

Hardwares e Softwares em Servidores

(Voltar)

Muito semelhante a um computador. Quando ocorre uma requisição HTTP (o navegador client pede informações do servidor), o sistema operacional executa o programa de Servidor Web para a consulta e o envio da informação solicitada.

Tipos de Servidores

(Voltar)
  • Arquivos
  • Segurança: um servidor de proxy é um servidor adicional que fica no meio do caminho entre o client e a internet. Ele estabelece regras para ver se o usuário pode ou não acessar determinado site. Um firewall protege contra acessos externos maliciosos, ele se encontra no mesmo espaço do proxy, mas atua em um cenário diferente.
  • Streaming
  • E-mail
  • Web
  • Database Server
  • Servidor DNS: o DNS traduz o nome em um IP específico (o endereço do site dentro da internet, baseado no nome dele).

Hospedagem de sites

(Voltar)

Primeiro, contratar um servidor de hospedagem e registrar um domínio (meusite.com.br) para o meu site (a empresa contratada faz isso pra mim?). A conexão entre client e server envolve o FTP (File Transfer Protocol), que é uma conexão com o servidor para a transferência de arquivos.


Linguagens de Programação

O que é

(Voltar)

Linguagem escrita e guiada por um conjunto de regras, com o objetivo de gerar softwares.

Server-side

(Voltar)

Linguagens que são executadas no lado do servidor: PHP, C#, Ruby, JavaScript, Java... elas farão toda a lógica e processamento necessários para que o servidor sirva às requisições.

Client-side

(Voltar)

Linguagens que rodam no cliente (na máquina do usuário), no que se trata de aplicações web. A grosso modo, o navegador roda HTML, CSS e JavaScript. Por que o JavaScript roda tanto no client quanto no server? Por causa do V8, que é o módulo de execução, o mecanismo que de fato roda a linguagem. No lado do server, o JS anda em conjunto com o Node.JS, já no client, é o navegador que atua como plataforma de execução (ambos com o V8 embutido).

HTML Não é Linguagem de Programação

(Voltar)

É, na verdade, um arquivo de marcação do texto, que define as características visuais, como negrito, itálico, sublinhado, limites de parágrafos e outras funções específicas, aplicadas pra cada parte do conteúdo, dentro de limites. Este parágrafo, por exemplo, tem a tag "p" abrindo e fechando o seu texto. Essa frase tem a tag "b" abrindo e fechando ela dentro do código.


Introdução ao HTML na Prática

Ferramentas Utilizadas

(Voltar)

Vamos codar usando o Visual Studio Code (mais chamado de VS Code), plataforma open-source em que escrevo nesse exato momento, e é, de longe, a mais utilzada para codificar, ao menos até onde me diz minha experiência.

Inspetor de Elementos

(Voltar)

Muito bom para estudo do HTML que estrutura páginas alheias. Dá até pra manipular os elementos exibidos através do console, mas não é nada que vai ser registrado no servidor, ou seja, vai aparecer só pra mim, e assim que eu atualizar a página, ela já volta a ser exibida como foi originalmente construída.

Estrutura Básica do HTML

(Voltar)

Para usar o VS Code, é definida uma pasta (em algum local do computador) que contém um arquivo dentro dela. No caso do HTML, é padrão que o arquivo principal seja chamado de "index.html". A estrutura básica do HTML começa com uma tag chamada "html", seguida do "head" (informações que serão carregadas antes de exibir a página) e o "body" (o que será, de fato, exibido para o usuário).

Falando sobre Tags

(Voltar)

Elas definem o tipo de conteúdo que aparece entre seu início e fim. É um parágrafo? Uma quebra de linha? Negrito? Itálico? Um bloco de conteúdo?

A maioria das tags "abraça" um trecho de conteúdo para modificá-lo. Por exemplo, se quero que esta frase esteja em negrito, Eu abro a tag "b" antes do início da frase, e fecho essa tag ao final da frase.

Algumas tags, como o "input", fogem à essa regra, pois não precisam do fechamento. O input, no caso, faz surgir um elemento de entrada de dados, que pode ser de texto, número, etc.

Atributos das Tags

(Voltar)

São propriedades que uma tag pode ter. É um detalhamento maior sobre o que a tag faz, e como ela vai se comportar, qual o escopo de atuação dela naquela situação específica...

Alguns exemplos:

  • Atributos genéricos:
  • "id": um identificador dos elementos. Vai ser muito bom para quando o JavaScript entrar em jogo, já que ele consegue ler o HTML e fazer modificações baseado nas IDs.
  • "style": uma maneira de formatar o HTML, trazendo funções do CSS. Pode ser usado diretamente nos elementos
  • "class": um comando que padroniza elementos. Se eu quiser que todos os subtítulos da página sejam azuis, posso criar uma classe que define isso, e depois chamar essa classe quando escrevo os subtítulos.
  • Atributos específicos:
  • "number" e "text": são aplicados à tag input, e definem o tipo de entrada de dados no campo que a pessoa vai digitar como número e texto, respectivamente.
  • "src": define o local de origem de algo que quero exibir na minha página, como uma imagem, por exemplo. Ao escrever código no VS Code, posso salvar o arquivo na mesma pasta que estou codando, e definir o "src" (source) como o nome da imagem desejada.

Textos

(Voltar)

Tags específicas para formatação de textos:

  • Tags "h": São tags de título em diferentes níveis. Quanto menor o número após o "h", maior a posição em hierarquia do título ou subtítulo em questão;
  • Tag "p": Serve para delimitar um parágrafo;
  • Tag "blockquote": Destaca um trecho de texto que é uma citação externa.
  • Tag "b": Negrito;
  • Tag "i": itálico;
  • Tag "mark": efeito de marca-texto;
  • Tag "sup": coloca um elemento um pouco acima do texto normal, como um número ao quadrado. (Exemplo: 22);

Listas Ordenadas e Não Ordenadas

(Voltar)

Existem basicamente dois tipos de listas no HTML: as ordenadas ("ol") e as não ordenadas ("ul"). Não dá pra adicionar conteúdo a elas diretamente, é preciso aplicar tags filhas ("li") para cada item da lista.

Lista ordenada:

  1. Item 1
  2. Item 2
  3. Item 3

Lista não-ordenada:

  • Item 1
  • Item 2
  • Item 3

Links

(Voltar)

Links (tag "a"), no HTML, podem levar tanto para uma página externa (como de costume) quanto para algum ponto dentro da mesma página. Os dois destinos precisam do atributo "href" para serem definidos.

  1. Para páginas externas: adicionar a URL desejada como o valor do atributo "href";
  2. Para outro ponto da mesma página: adicionar a id do elemento desejado ao atributo "href". A página vai fazer um scroll automático até o elemento referenciado.
